GIT

Desarrollo Colaborativo

Areas de GIT

Working Directory

Es el area de trabajo, donde rara vez vamos a utilizar la linea de comandos. Aquella donde realizamos las modificaciones de nuestro proyecto, tales como la creacion, edicion y eliminacion de archivos. El unico comando necesario sera git init para iniciar nuestro proyecto gestionado con la herramienta git.

Staging Area (INDEX)

Es el area de control, donde agregamos los archivos a los que queremos hacerle seguimiento, una vez hecho esto podemos realizar la confirmacion de cambios para poder recuperar dichas modificaciones, en caso de ser necesario. Cuando agregamos un archivo al INDEX git realiza una captura del codigo (SNAPSHOT) que permite identificar las lineas agregadas o eliminadas.

Repository(Local/Remote)

Area donde se almacenan las confirmaciones de las captura en formato BLOB(Binary Large Object File) para que puedan ser utilizadas al momento de movernos en nuestro proyecto y reestablecer el mismo. Git gestiona los cambios localmente, pero podemos enviar el historial de cambios a una direccion externa, tambien conocida como repositorio remoto.

Apuntadores de GIT

HEAD

Apuntador movil que indica al usario en que punto del historial de cambios se encuentra. Normalmente de deplaza conjuntamente con la rama en la que nos encontremos y podemos utilizarlo como referencia al movernos dentro del historial, teniendo en cuenta que los commits solamente identifican a sus ancestros.

BRANCH

Apuntador dinamico que siempre se corresponde con el ultimo commit de una linea de trabajo. Facilitan la navegacion, porque nos permiten volver al ultimo commit debido a que estos solamente identifican a sus ancestros y por lo tanto no reconocen cuales son las confirmaciones posteriores. Se utilizan para implementar o probar caracteristicas, sin comprometer la linea principal del proyecto.
